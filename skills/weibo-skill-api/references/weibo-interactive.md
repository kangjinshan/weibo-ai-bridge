# 微博互动内容分析

帮助博主发现最值得回复的评论，促进与粉丝之间的真实互动。通过分析评论者的关注关系和认证状态，智能排序推荐优先回复的评论，让每一次互动都更有价值。

---

## 适用场景

- 博主想知道"哪些评论值得优先回复"
- 博主想了解"谁在关注我、谁是大V"
- 博主想提升粉丝互动率，维护核心粉丝关系
- 博主想查看某条微博下的完整评论列表

---

## 命令说明

### 1. 获取收到的评论列表

获取博主收到的所有评论，包含评论者与博主的关注关系信息：

```bash
node scripts/weibo-skill.js interactive-comments-to-me
```

### 2. 获取自己某条微博的所有评论

查看自己指定微博下的完整评论列表：

```bash
node scripts/weibo-skill.js interactive-comments-show --id=5305687166161572
```

**参数说明**：

| 参数 | 必填 | 说明 |
|------|------|------|
| `--id` | 是 | 微博 ID（博文 ID），注意不是评论 ID |

---

## 返回数据结构

两个命令返回结构一致：

```json
{
  "code": 0,
  "data": {
    "comments": [
      {
        "createdAt": "Mon Jun 15 16:14:47 +0800 2026",
        "id": "评论ID",
        "mid": "评论mid",
        "postJumpLink": "https://weibo.com/<uid>/<mid>",
        "text": "评论内容...",
        "user": {
          "followMe": false,
          "following": false,
          "id": "用户ID",
          "name": "用户昵称",
          "verified": false
        }
      }
    ],
    "count": 1,
    "msg": "查询成功"
  },
  "message": "success"
}
```

### 关键字段说明

**评论信息**：

| 字段 | 说明 |
|------|------|
| `text` | 评论内容 |
| `createdAt` | 评论时间 |
| `postJumpLink` | 原博跳转链接，可直接点击查看上下文 |

**用户关系信息**（这是互动分析的核心数据）：

| 字段 | 含义 | 互动价值 |
|------|------|----------|
| `followMe=true` | 对方是我的粉丝 | 回复粉丝能增强粘性 |
| `following=true` | 我关注了对方 | 回复关注的人能维护关系 |
| 两者都为 `true` | 互粉关系 | 最值得互动的核心用户 |
| `verified=true` | 认证用户（大V） | 回复大V能提升曝光 |

---

## 注意事项

1. 需要先配置好 `weibo-ai-bridge` 的微博凭证；脚本会自动读取或刷新 Token
2. `postJumpLink` 格式为 `https://weibo.com/<uid>/<mid>`，可直接跳转到原博
3. 关注关系反映的是查询时刻的状态，可能随时间变化
4. `interactive-comments-show` 的 `--id` 是微博 ID（博文 ID），不是评论 ID

---

## 数据分析能力：互动回复建议

当用户询问"哪些评论值得回复"、"如何优先回复粉丝"、"互动建议"等问题时，按以下流程进行分析。

> ⚠️ **分析报告必须遵守以下规则**：
> 1. **报告中提及的每一条评论，都必须附带该评论对应的原博链接**（`postJumpLink`），无论出现在哪个部分，确保用户可以直接点击跳转查看原博和评论上下文。不允许出现没有链接的评论提及。**原博链接必须使用 Markdown 链接格式** `[url](url)` 输出。
> 2. 每条评论必须给出 **AI 建议的回复示例**（用 `💬 AI回复示例：` 标识），根据评论内容、关系类型和认证状态量身定制，语气自然友好，供用户参考或直接复制使用。
> 3. 每条评论必须给出**简短的回复思路说明**（用 `📝 回复思路：` 标识），解释 AI 回复示例为什么这么写——即回复策略和措辞背后的考量，一句话即可。

---

### 核心思路

不是所有评论都值得同等对待。互粉好友的评论、忠实粉丝的留言、大V的互动——这些都比路人评论更值得博主花时间回复。通过关注关系和认证状态，帮博主找到最该回复的那几条。

### 优先级排序规则

| 优先级 | 关系类型 | 基础分 | 认证加成 | 说明 |
|--------|----------|--------|----------|------|
| 最高 | 互粉（`followMe` + `following`） | 30 | +5 | 核心社交圈，必须维护 |
| 高 | 粉丝（仅 `followMe`） | 20 | +5 | 忠实受众，回复能增强粘性 |
| 中 | 我关注的（仅 `following`） | 10 | +5 | 博主主动关注的人，有社交价值 |
| 低 | 无关系 | 0 | +5 | 路人评论，选择性回复 |

> 分数相同时，按评论时间降序（最新优先）。默认推荐前 **5** 条，最多不超过 **20** 条。

### 分析步骤

**第一步：获取数据**

```bash
# 获取所有收到的评论
node scripts/weibo-skill.js interactive-comments-to-me

# 或获取自己某条微博的评论（用户指定了微博时）
node scripts/weibo-skill.js interactive-comments-show --id=<weibo_id>
```

**第二步：分类统计**

将评论按关注关系分为四类，同时标记认证用户。

**第三步：计算优先级并排序**

按上述评分规则计算每条评论的优先级分数，降序排列。

**第四步：输出分析报告**

### 报告结构

> ⚠️ 报告中每条评论的原博链接**必须使用 Markdown 链接格式** `[url](url)` 输出，方便用户点击跳转。
